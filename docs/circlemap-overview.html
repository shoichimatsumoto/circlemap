CircleMap 全体像ガイド

作成日: 2026年6月21日 ／ 対象: circlemap-five.vercel.app

CircleMap は何をしているサイト？

FANZA の同人作品を、サークル（制作団体）単位で探しやすく見せるサイトです。

  • 作品一覧
  • サークル一覧
  • 検索
  • 作品詳細（購入リンク付き）

登場する「3つの場所」

イメージは 「本屋」 です。

場所 何？ 例え
DMM API FANZA の公式データ窓口 出版社の倉庫（本はあるが遠い・取りに行くのに時間がかかる）
Supabase（DB） 自分用のデータ置き場 店舗の棚（一度並べておけば、お客さんにすぐ見せられる）
Vercel サイト本体（CircleMap） お店の店舗（お客さんがブラウザで見るところ）

2種類の「データの流れ」

「見せる」と「更新する」は別ルート です。ここが全体像の核心です。

流れA：お客さんがサイトを見る（いつも）

お客さんが circlemap-five.vercel.app を開く
        ↓
Vercel 上の CircleMap が動く
        ↓
Supabase の棚（DB）から作品・サークルを読む  ← Step 5 でこうなった
        ↓
画面に表示（速い）

お客さんは DMM API を直接叩いていません。棚（Supabase）にあるデータを見ているだけです。

流れB：棚の中身を最新にする（たまに／これからは毎日）

あなた（または自動 Cron）が /api/sync を実行
        ↓
「合言葉（SYNC_SECRET）」が合っていれば OK
        ↓
CircleMap が DMM API から最新データを取得
        ↓
Supabase の棚に上書き保存

/api/sync は「棚を補充する裏口」です。一般の訪問者向けの入口ではありません。

図でまとめると

【お客さん向け】毎日何百回も起きる
  ブラウザ → CircleMap（Vercel）→ Supabase（棚）→ 画面表示
                                    ↑
                                    速い・安定

【管理者向け】1日1回くらい（Step 6 で自動化予定）
  あなた or Cron → /api/sync?secret=合言葉
                        ↓
                   DMM API（倉庫）から取得
                        ↓
                   Supabase（棚）を更新

左（表示）と右（同期）は別物 です。

環境変数は何のため？

Vercel に入っている設定の「鍵・住所」です。

変数名 役割（ざっくり）
DMM_API_ID など 倉庫（DMM）に入るための鍵
NEXT_PUBLIC_SUPABASE_URL など 棚（Supabase）の住所
SYNC_SECRET 裏口（/api/sync）の合言葉
  • DMM の鍵 → 同期するときだけ使う
  • Supabase の住所 → 表示するときも同期するときも使う
  • SYNC_SECRET → 同期の裏口だけの合言葉

Step 1〜6 は何をやってきたか

Step やったこと 例え
1 Supabase プロジェクト作成 棚を用意した
2 テーブル作成 棚に「作品」「サークル」の区画を作った
3 接続確認 店舗から棚に届くか確認
4 初回同期 倉庫から初めて本を棚に並べた（355サークル・450作品）
5（完了） 表示を棚から読む お客さんは倉庫に行かず、棚を見るだけに
6（これから） 毎日自動同期 毎朝、裏口から棚を自動で補充

Step 5 までで「速く見せる仕組み」は完成しています。Step 6 は「棚を自動で最新に保つ」追加です。

SYNC_SECRET（裏口の合言葉）について

サイトの表示 → 関係なし（Supabase を読むだけ）

同期（/api/sync）→ 合言葉が必要

合言葉がチャットに出ていたので、「裏口の鍵を新しいものに差し替えよう」という話でした。 本を並べ替える作業（Step 5）とは別の、セキュリティの話 です。

今すぐ変えなくて大丈夫？

はい、今すぐでなくて大丈夫です。

  • サイトの表示には影響しません
  • 裏口を知っていても、サイトを壊したり決済情報が漏れたりはしません
  • 最悪の場合でも「知らない人が同期を何度も実行する」程度のリスクです

おすすめのタイミング：

  • Step 6（自動同期）を設定するとき — そのタイミングで一緒に変えると効率的
  • または、余裕があるときに Vercel で変更 → Redeploy

いまの状態（整理）

  • ✅ 棚（Supabase）にデータがある（450作品など）
  • ✅ サイトは棚から表示する（Step 5）
  • ⏳ 棚の自動更新（Step 6・Cron）はまだ
  • ⏳ SYNC_SECRET の変更は推奨（任意・後回し可）

一言で言うと

CircleMap = 店舗（Vercel）

Supabase = 商品棚

DMM = 仕入れ元の倉庫

/api/sync = 棚を補充する裏口（合言葉付き）

お客さんは 棚だけ 見ています。あなた（または Cron）だけが 裏口から棚を更新 します。

主要 URL

本番サイト https://circlemap-five.vercel.app
状態確認 https://circlemap-five.vercel.app/api/status
DB 接続確認 https://circlemap-five.vercel.app/api/supabase/status
手動同期（合言葉必要） https://circlemap-five.vercel.app/api/sync?secret=合言葉
